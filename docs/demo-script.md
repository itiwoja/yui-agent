## デモの流れ（5〜7分 / 1日のストーリー仕立て）

### 0. 準備（画面に出す前に）

- ブラウザで本番URLを開き「そばに置く」を ON（マイク許可済みの状態にしておく）
- 別タブ: ダッシュボード / GitHub Actions / スマホに Google Tasks アプリ

### 1. 朝 — 独り言がタスクになる（約1分）

- 画面に向かわず、作業しながら独り言:
  - 「あー、**Webデザインの課題**、金曜までだった気がする…」
- → 呼びかけ無しなので**ゆいは黙ってタスク化**（/process 経路）
- ダッシュボードを開いて「言っただけのことが拾われている」を見せる

### 2. 昼 — 呼びかけて相談（マルチツールの見せ場・約2分）

- 「**ゆい**、今日なにからやればいい？」
- → ゆいが **Google Calendar の今日の予定＋Firestore の未完了タスク優先度** を合わせて提案
  （例: 「15時から授業だよね。その前に課題の下書きやっちゃお？」）
- 続けて（45秒の会話ウィンドウ内なので名前なしで）:
  - 「そういえば副業の**請求書**も出さなきゃなんだよね」→ 会話からタスク追加

### 3. 裏側 — エージェントが勝手に働く（約1.5分）

- `POST /autonomous-review` を手動トリガー（本番は Cloud Scheduler が30分毎）
- ダッシュボードで見せる:
  - 繰り返し言及されたタスクの**優先度エスカレーション**（🔴🟠🟡）
  - Google Search grounding による**調査ノート**
  - 「**ゆいから質問があります**」= 保留質問（needs_input）
- スマホの Google Tasks にも同期されていることを見せる

### 4. 夕方 — 「終わったよ」でループが閉じる（約1分）

- 「ゆい、**Webデザインの課題終わったよ**」
- → ゆいがねぎらう＋Firestore と Google Tasks の**両方が完了**になる
- ダッシュボードの完了一覧で確認

### 5. 締め — 「まわしている」を見せる（約1.5分）

- GitHub Actions のタブ: **push → check（依存解決+構文検証）→ Cloud Run デプロイ** のパイプライン
- Cloud Run のリビジョン履歴（自動デプロイの証跡）
- アーキテクチャ一言まとめ:
  - Cloud Run + Vertex AI (Gemini 2.5 Flash) + Speech-to-Text + TTS (Chirp3-HD) + Firestore + Cloud Scheduler + Secret Manager + WIF キーレスCI/CD

## 想定問答

**Q. なぜ ADK（Agent Development Kit）を使わなかった？**
A. エージェントの中核が「Sense→Diagnose→Act/Ask」という確定的な制御フローで、Gemini の構造化出力＋明示的な分岐で書く方がデバッグ容易性と挙動の説明可能性が高いと判断しました。ツールが増えるフェーズ（Calendar 書き込み・Gmail 等）で ADK のツール管理・セッション管理に載せ替えるのがロードマップです。

**Q. プライバシーは？ 常時マイクは怖くない？**
A. 音声はブラウザ内で VAD 判定し、発話区間のみサーバーへ送信。閾値・無音判定はクライアントで調整可能です。データは本人の Firestore / Google Tasks にのみ保存され、OAuth トークンは Secret Manager 管理、Cloud Run はランタイムSAの最小権限で動きます。

**Q. 呼びかけの認識精度は？**
A. STT の phrase hints（boost=15）＋表記ゆれを許容する照合、一度呼びかけた後45秒は名前なしで会話継続、という2段構えです。無音判定などの環境依存パラメータは URL クエリで会場に合わせて即調整できます。

**Q. 既存のタスクアプリ・Circleback 等との違いは？**
A. 既存ツールは「入力されたもの」を管理します。ゆいは「入力されなかったもの」を発見し、優先度を育て、質問し、完了まで見届けます。管理ツールではなく秘書です。

## リハーサル・チェックリスト（本番同環境で最低3回）

本番と同じ PC・マイク・ネットワーク・会場の騒音条件で通しリハーサルする。

- [ ] リハ1回目: 素の状態で台本を通す。呼びかけ漏れ・誤検出・無音判定の切れ方をメモ
- [ ] パラメータ調整: `?silence=1800&threshold=12` のように URL クエリで調整して再試行
  - `silence` (既定2200ms): 早く切れる→上げる / テンポが悪い→下げる
  - `threshold` (既定10): 騒がしい会場→上げる / 拾いが悪い→下げる
  - `minSpeech` (既定500ms) / `maxRecord` (既定20000ms) / `conversation` (既定45000ms)
- [ ] リハ2回目: 調整後の値で通す。TTS再生中にオーブタップで停止→復帰も1回試す
- [ ] リハ3回目: 決定値で最初から最後までノーミスで通す
- [ ] 決定値をコードのデフォルト値に反映して push（→ CI/CD で自動デプロイ）
- [ ] 当日朝: `/autonomous-review` の手動トリガーと Google Tasks 同期を最終確認
- [ ] 保険: デモが崩れた場合に見せる録画（リハ3回目を画面収録しておく）
